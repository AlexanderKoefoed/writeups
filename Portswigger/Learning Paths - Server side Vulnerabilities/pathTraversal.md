# Path Traversal

Path Traversal (or Directory traversal) is the action of discovering paths on a server which can be freely read or accessed. Common examples include:

- Application code and Data
- Credentials for back-end systems (Commonly hardcoded in the source of a website)
- Sensitive operating system files

In extreme cases, the attacker may be able to write arbitrary files on the server, allowing the attacker to modify data and/or behavior of the server. This can lead to a full takeover of the server.

An example is given together with a lab, where we are tasked with retrieving the contents of the /etc/passwd file of the server.

## Lab: File path traversal, simple case

**Lab Description**:  This lab contains a path traversal vulnerability in the display of product images. To solve the lab, retrieve the contents of the `/etc/passwd` file.

### Lab Solution

It is seen that the webshop presented in the lab, fetches the images for items through a path with the following query param:

``` HTML
<img src="/image?filename=11.jpg">
```

This suggest that the filename is retrieved from a folder located on the webserver through a filesystem API. The server is not guarded against attacks through this query, meaning the attacker can request arbitrary files on the server by guessing the correct folder and file name. 
To obtain the `/etc/passwd` file we request the resource at `/image?filename=../../../etc/passwd` and the server returns the contents of the passwd file:
